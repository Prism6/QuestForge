## 2. 발견된 OOP 위반 사항

### 2-1. `utils/prompts.py` — 모듈 함수 + 지역변수 딕셔너리

| 원칙 | 위반 내용 |
|------|-----------|
| **OCP** (개방-폐쇄 원칙) | `genre_tone_guide`, `quest_type_ko_to_en` 딕셔너리가 함수 내부 지역변수로 정의. 장르/타입 추가 시 함수 내부를 직접 수정해야 함 |
| **DRY** (중복 배제) | `quest_type_ko_to_en`이 `prompts.py`에 한 번, `app.py`에 역방향(`QUEST_TYPE_KO`)으로 다시 정의 — 동일 정보가 두 곳에 분산 |
| **성능** | 함수 호출마다 딕셔너리 객체를 새로 생성 |

### 2-2. `utils/quest_generator.py` — 매직 상수 하드코딩

| 원칙 | 위반 내용 |
|------|-----------|
| **SRP** / 가독성 | `"claude-sonnet-4-20250514"`, `2000`, `1.0` 이 메서드 내부 호출부에 직접 기재 |
| **OCP** | 모델·파라미터 변경 시 메서드 내부를 수정해야 함 |
| **재사용성** | `validate_quest_data()` 내 `required_fields` 리스트가 호출마다 생성되는 지역변수 |

### 2-3. `utils/data_exporter.py` — 메서드 내부 하드코딩

| 원칙 | 위반 내용 |
|------|-----------|
| **OCP** | `column_order` 리스트가 `to_excel()` 내부에 정의 — 컬럼 추가 시 메서드 내부 수정 필요 |
| **가독성** | `min(max_length, 50)` — 매직 넘버 50 |
| **OCP** | MIME 타입 문자열, 시트 이름 `'Quests'`가 `create_download_data()` 내부에 분산 |
| **검증 시점** | 지원 형식 검증이 조건 분기 마지막에서야 이루어짐 |

### 2-4. `app.py` — 중복 정의 + 하드코딩 옵션 목록

| 원칙 | 위반 내용 |
|------|-----------|
| **DRY** | `QUEST_TYPE_KO = {"main": "메인", ...}` — `prompts.py`와 역방향 중복 |
| **OCP** | 사이드바 장르 목록 `["RPG", "액션", ...]`이 인라인 하드코딩 — `prompts.py` 장르 추가 시 `app.py`도 수정 필요 |

## 3. 수정 내용

### 3-1. `utils/prompts.py` — `PromptBuilder` 클래스 도입

**변경 전:** 모듈 수준 함수 2개 (`create_quest_prompt`, `create_regeneration_prompt`)
**변경 후:** `PromptBuilder` 클래스 + 클래스 상수 3개 + 클래스 메서드 2개

```python
class PromptBuilder:
    GENRE_TONE_GUIDE: Dict[str, str] = { ... }        # 장르별 톤 가이드
    QUEST_TYPE_KO_TO_EN: Dict[str, str] = { ... }     # KO → EN 매핑
    QUEST_TYPE_EN_TO_KO: Dict[str, str] = {           # EN → KO (자동 파생)
        en: ko for ko, en in QUEST_TYPE_KO_TO_EN.items()
    }

    @classmethod
    def create_quest_prompt(cls, ...) -> str: ...

    @classmethod
    def create_regeneration_prompt(cls, ...) -> str: ...
```

- `QUEST_TYPE_EN_TO_KO`는 `QUEST_TYPE_KO_TO_EN`을 자동으로 역방향 파생 → 단일 출처(SSOT) 확보
- 장르/타입 추가 시 `GENRE_TONE_GUIDE` 또는 `QUEST_TYPE_KO_TO_EN` 상수만 수정하면 앱 전체 반영

### 3-2. `utils/quest_generator.py` — 클래스 상수 추출

**변경 전:** 인스턴스 변수 `self.model`, 인라인 `max_tokens=2000, temperature=1.0`, 지역변수 `required_fields`
**변경 후:** 클래스 상수 5개 추출

```python
class QuestGenerator:
    MODEL: str = "claude-sonnet-4-20250514"
    MAX_TOKENS: int = 2000
    TEMPERATURE: float = 1.0
    REQUIRED_FIELDS: List[str] = ["quest_name", "quest_type", ...]
    NESTED_DICT_FIELDS: List[str] = ["npc", "objective", "rewards", "dialogue"]
```

- `self.model` → `self.MODEL` (클래스 상수 참조): 인스턴스별 오버라이드 없는 고정값을 명시적으로 클래스 수준으로 이동
- `PromptBuilder.create_quest_prompt()` / `create_regeneration_prompt()` 로 임포트 경로 갱신

### 3-3. `utils/data_exporter.py` — 클래스 상수 추출 + `_flatten_quest` 분리

**변경 전:** `column_order`, MIME 문자열, `50`, `'Quests'`가 메서드 내부에 분산
**변경 후:** 클래스 상수 4개 + 내부 헬퍼 메서드 분리

```python
class DataExporter:
    COLUMN_ORDER: List[str] = [...]          # 컬럼 순서
    MAX_COLUMN_WIDTH: int = 50               # 최대 컬럼 너비
    SHEET_NAME: str = "Quests"              # 시트 이름
    MIME_TYPES: Dict[str, str] = {          # MIME 타입 매핑
        "json": "application/json",
        "excel": "application/vnd...."
    }

    @staticmethod
    def _flatten_quest(quest: Dict) -> Dict: ...  # 중첩 구조 평면화 책임 분리
```

- `create_download_data()`에서 지원 형식 검증(`if format_type not in cls.MIME_TYPES`)을 앞으로 이동
- 중첩 퀘스트를 평면화하는 로직을 `_flatten_quest()` 로 분리하여 SRP 강화
- `to_excel()` → `@classmethod` 로 변경 (클래스 상수 참조)
- 반환 타입 힌트 `tuple` → `Tuple[bytes, str, str]` 명시

### 3-4. `app.py` — 중복 제거 + SSOT 적용

**변경 전:**
